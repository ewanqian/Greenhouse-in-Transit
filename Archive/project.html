项目说明

这件作品从一只银色手提箱开始，但目标不是展示"箱子"本身，而是展示一个箱子如何被理解为一座可被运输、被误投、被重新开启的空间。它表面像旅居系统中的遗失行李，内部却封存了一段植物性记忆环境。

观者在现实中先看到真实箱体，在网页中进入赛尼亚岛数字旅居地与相关机构档案，在 Vision Pro 中则会看到一段短暂展开的温室空间：边界球壳、线框结构、漂浮代码、点云植物、高斯花园残影与局部环境参数从箱内显影，形成一次近身而克制的 MR 剧场。

为什么是它

这个项目同时解决几件事：一是把赛尼亚岛数字旅居地从"地图与概念"变成一件可落地的现实证物；二是把 Apple Vision Pro 从展示工具变成叙事入口；三是把网页、地图、Minecraft、机构物料、票据、小礼品和 ARG 线统一到一件具体物件上。

这套结构也非常适合接入"艺术航站楼 / 登机仪式"的展演环境：统一手提箱媒介、开幕时携带作品到场、允许表演和交换等条件都非常匹配这一项目的入口性与过境性。

世界观总览

表层

赛尼亚岛数字旅居地是一座模拟挪威峡湾景观的虚拟北欧目的地，由"松涧目的地"作为旅居品牌对外运营。用户可以看地图、看物料、进入 Minecraft 游玩、理解空间节点与旅居方式。
深层

这座岛屿同时属于更大的数字航行宇宙。旅客和维护员通过数据洋流潜航系统在目的地节点之间迁移，并携带便携空间容器、记忆仓与环境培养舱。
异常

《箱中温室》原本是一位长期驻留旅客的便携温室舱。在一次复制或中转错误中，它错误脱离系统，出现在现实世界，成为展览现场被观者捡到的一件遗失物。

视觉方法

这个项目的视觉不会只做成唯美花园，也不会只做成科技 demo。它更接近"旅居品牌视觉 + 内网档案系统 + 复古数字温室"的混合体：北欧自然、植物湿度、线框结构、漂浮代码、Y2K 数字感、虹彩圆盘和高斯植物共同构成其识别度。

体验流程

01 / 遇见
现实入口
观众先在现场看到一只真实手提箱，以及关于赛尼亚岛数字旅居地和遗失物的简短说明。第一层理解是"这是一个来自虚拟目的地的东西"。
02 / 扫码
项目主页
扫码后进入本页，快速理解项目、世界观、赛尼亚岛、组织结构与体验路径，确认这并非单一装置，而是一整套数字空间与现实侵入系统的入口。
03 / 深入
档案查看器
观众进一步进入数据库式页面，查看组织、地点、物件、旅客和异常记录，开始意识到"箱子来自一个更大的系统"。
04 / 显影
Vision Pro
佩戴 Vision Pro，观看 60–90 秒的箱体展开动画：球壳、线框、点云植物、代码和高斯花园残像从箱内短暂显影。
05 / 留存
线下物件
观众返回现实箱体，查看票据、标签、环境参数卡、旅客用品、气味卡与其他遗失物。此时现实中的箱子会被重新理解为"一个被捡到的世界观证物"。

松涧目的地

数据洋流潜航署
维护数字洋流中的航行秩序、目的地通行与旅客中转。
空间转换机构
负责空间容器、错误复制物与遗失系统的回收与登记。

前期创作脉络

新媒体艺术家模拟器

以模拟器和伪系统界面处理职业、角色与媒介现实，为本项目的"系统外壳 + 深层叙事"提供方法基础。
CURATOR

以社交产品界面为壳，逐步暴露关系张力与误读机制，为本项目的 ARG 页面、档案查看器与假网页模拟提供经验。
工作坊 / Vision Pro / 数字花园实验

提供空间计算、植物记忆、网页原型和展示逻辑的前期测试，使《箱中温室》能够兼顾艺术项目与研究样本两种身份。
它不是一个介绍"未来科技"的项目，而是一件把空间、旅居、机构、记忆与错误投递同时装进现实的入口物件。
