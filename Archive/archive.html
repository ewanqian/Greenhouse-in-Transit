RULE-001 Public Access

数字洋流

基础规则

数字洋流是整个 VIRTURA 宇宙中最基础的流动媒介。

它既像网络，又像海流；既像数据迁移通道，又像空间传输介质。不同空间节点、驻泊地、温室舱、旅居目的地和中转容器，都会通过数字洋流建立连接。

所有空间的生成、复制、运输、丢失都发生在数字洋流之中。

RULE-002 Public Access

潜航系统

基础规则

为了在数字洋流中长期稳定航行，一套潜航系统被建立起来。

负责范围：

  • 空间节点之间的迁移
  • 旅客与维护人员的通行
  • 便携空间容器的运输
  • 环境样本与记忆片段的封存
  • 异常事件的回收与登记

潜航系统维持着整个数字航行网络的秩序。

RULE-003 Public Access

目的地节点

基础规则

某些空间被开发、维护并稳定下来，成为可驻留、可游玩、可体验的目的地节点。

这些节点表面上可被理解为虚拟景区、旅居地、实验空间或沉浸式岛屿，深层上则是数字洋流系统中的稳定港口。

赛尼亚岛数字旅居地就是其中之一。

RULE-004 Public Access

便携空间容器

基础规则

在这套宇宙中，空间并不总是固定存在。

有些空间被压缩、封存、编码后，可以装入便携式容器。这些容器可能表现为手提箱、设备箱、收纳舱、培养仓、珠宝盒、档案匣等。

它们不是普通器物，而是"小型空间"的物质形态。

RULE-005 Public Access

复制错误与异常投递

基础规则

由于空间复制、中转协议、临时缓存和环境封存过程中的不稳定性，某些容器可能在错误时间、错误地点或错误维度中出现。

当这种情况发生时，现实世界会短暂接收到来自数字宇宙的遗失物。

ARG 式的侵入现实事件，即由此产生。

箱中温室就是一个典型案例。

LOC-001 Public Access

赛尼亚岛数字旅居地

目的地节点

表层定义：赛尼亚岛数字旅居地是一座模拟挪威峡湾景观的虚拟北欧旅居岛屿。它被设计为一处能够长期驻留、游玩、探索与感受的目的地，融合北极圈地貌、峡湾气候、北欧岛屿风景与东方静谧美学。

深层定义：在更深层的世界观中，赛尼亚岛不是一座普通岛屿，而是数字洋流中的一个驻泊港。它相对稳定、风貌清晰、环境宜居，因此成为旅客、维护员、研究型乘客和长期停靠者愿意选择的目的地之一。

当前公开入口：赛尼亚岛目前最公开、最低门槛的入口是 Minecraft 版本。用户可以作为普通游客或玩家进入岛屿，沿地图中标注的路径、村落、山海与旅居建筑进行探索。这使赛尼亚岛具备真实可游玩的特性，而不是停留于概念说明。

后续可扩展入口：

  • 网页导览版本
  • 高斯重建版本
  • Apple Vision Pro 近身体验版本
  • 角色化导览版本
  • ARG 失物入口版本

ANI-001 Public Access

箱中温室

异常物 / 便携空间容器

定义：箱中温室是一件来自赛尼亚岛宇宙中的便携温室培养舱。它外观上是一只手提箱，内部却储存着植物性记忆、环境样本、局部温室结构和一段被压缩的驻留空间。

滴流项目早期概念图 - 数字自然与几何结构的探索

原属关系：它原本属于一位在赛尼亚岛长期驻留的旅客（身份待确认），或研究型停泊者。该旅客将自身在冰岛、北方植物园、温室空间、旅行过程中采集到的图像、湿度、环境参数与局部空间感受整理、保存，并封存于这只便携温室舱中。

Blender → Vision Pro 3D空间视频工作流测试

异常事件：在一次空间复制或节点中转中，该箱体未能按预定路径回到赛尼亚岛某处驻留空间，而是错误脱离系统，进入现实世界。从此，它既是一件艺术作品，也是一件世界观证物，更是现实与数字宇宙发生接缝的破口。
