# 箱中温室 / Greenhouse in Transit

副标题：便携空间入口 / Portable Spatial Gateway

---

## 0. 文档用途

这份文档用于统一以下几件事：

1. 对外介绍项目时，能快速说明它是什么。
2. 为项目主页与档案查看器提供统一的信息源。
3. 为后续 GitHub 仓库、网页、ARG 页面、Vision Pro 原型、Minecraft 导览页提供一致的树状结构。
4. 为本地 agent、后续开发者和策展沟通提供可维护的数据库骨架。

---

## 1. 项目一句话

**《箱中温室》是一只被错误复制到现实世界的遗失手提箱。它属于赛尼亚岛数字旅居地宇宙中的便携温室舱；当箱体被打开，观者将短暂进入一段被封存的植物性记忆空间。**

### 2.1 对外定位

- 一个结合实体手提箱、Vision Pro 动画、网页档案与世界观设定的艺术项目

- 一个用于展示空间感知迁移、可携带空间、数字旅居宇宙与 ARG 设计方法的样本

### 2.2 内部定位

- 赛尼亚岛数字旅居地宇宙中的异常遗失物
- PineBrook / 松涧目的地体系中的边缘事件
- Spatial Transfer Bureau / 空间转换机构需要回收的一件错误投递物
- 后续网页、角色、地图、票据、气味、物件系统、手机端流程的母体

## 5. 项目叙事总纲

### 5.1 表层叙事

观众看到的是一个模拟挪威塞尼亚岛峡湾景观的数字旅居地项目。它拥有地图、旅居品牌、空间导览、建筑节点、游玩入口与精致的宣传系统，看上去像一个虚拟的北欧目的地。

### 5.2 深层叙事

这个目的地实际上属于更大的数字航行宇宙。旅客与维护员通过数据洋流潜航系统在不同目的地之间迁移，并携带便携空间容器、记忆舱、培养仓等设备。

### 5.3 异常事件

《箱中温室》原本是一位长期驻留旅客随身携带的便携温室舱，内部封存着一段植物性记忆环境。在一次空间复制或中转协议错误中，它被错误地复制并投递到现实世界，从而成为观众在展览现场“捡到”的遗失手提箱。

## 6. 体验流程总设计

### 6.1 线下总流程

```text
远看项目 → 看到手提箱 / 标识 → 读到项目简介 → 扫码 → 进入项目主页
→ 理解赛尼亚岛 / 松涧目的地 → 进一步进入档案查看器
→ 发现异常遗失物 → 理解箱中温室 → 进入 Vision Pro 观看
→ 回到现实物件 → 继续查看票据、标签、物件、礼品与气味
→ 产生“这是一个正在侵入现实的系统”的感受
```

### 6.2 最佳现场顺序

1. **第一眼**：看到真实箱子与项目标题
2. **第一层理解**：知道这是一个北欧数字旅居地宇宙中的物件
3. **第二层诱导**：扫码进入项目主页，看见地图、组织结构、赛尼亚岛导览
4. **第三层异常**：进入档案查看器，发现失物档案、机构记录、旅客线索
5. **第四层沉浸**：佩戴 Vision Pro，观看“箱体展开成温室空间”的短动画
6. **第五层留存**：拿走票据、标签、小礼品、气味卡或纪念物
7. **第六层延展**：回家继续通过网页追踪故事线

### 6.3 Vision Pro 单次体验节奏（建议 60–90 秒）

- 0–10 秒：箱体静置，轻微异常
- 10–20 秒：标签与锁扣苏醒，边界显影
- 20–40 秒：球形边界从箱体内部展开
- 40–65 秒：线框、点云植物、代码、虹彩圆盘与温室构架生长
- 65–85 秒：局部高斯花园记忆显影，短暂包裹观者
- 85–90 秒：空间回收，留下未认领状态

## 13. 当前版本结论

这个项目最合适的路线不是把所有内容一次做满，而是先用一只“遗失的便携温室箱”把整个宇宙打开。项目主页负责讲清楚，档案查看器负责讲深，线下箱体负责让它落地，Vision Pro 负责让空间真正显影。随后，赛尼亚岛、机构、乘客、票据、气味、物件与网页系统才会继续向外生长。
