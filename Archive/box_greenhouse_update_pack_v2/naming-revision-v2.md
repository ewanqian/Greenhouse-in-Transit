# 命名修正 V2

## 核心判断
上一版名称的问题不在“信息不够”，而在“语感太功能化”。很多名字像解释性 placeholder，而不像一个可信数字世界中真实存在的品牌、机构或装备。

## 最终采用

### 外层品牌
- **松间 / SONGJIAN**
- 角色：旅居品牌 / 岛屿外层公共叙述
- 理由：更像一个高端、安静、具有单名识别度的品牌，而不是“目的地公司”。

### 背后持有方
- **白屿会 / White Isle House**
- 角色：持有、开发、策划与长期维护母体
- 理由：避免“开发集团”的直接地产语感，保留一种优雅但有权力感的幕后存在。

### 航行网络
- **潮线 / TIDELINE**
- 角色：运输、接驳、航行与节点通行网络
- 理由：比“潜航署”更顺口，更像品牌化航运系统。

### 异常与回收部门
- **转栈署 / Relay Bureau**
- 角色：错投、失物、回收与归档
- 理由：比“空间转换机构”更凝练，也更适合出现在票据、警示语、网页表格与遗失记录中。

### 物件
- **项目名：箱中温室 / Greenhouse in Transit**
- **物件名：雾庭舱 / Mistgarden Case**
- **型号：MG-17 Portable Conservatory Unit**
- 理由：项目题名保留诗性，物件名补足装备级识别度。
