# 内容补强建议 V2

## A. 让赛尼亚岛不只是地图
- 在地图词条下补一页“温室模块 / Greenhouse Module”
- 说明赛尼亚岛不是只有旅居建筑，也有少量对公众不完全开放的保育与研究舱体
- 把雾庭舱和这些模块建立关系，增强可信度

## B. 让雾庭舱不只是一个箱子
至少补出 6 个相关附属物件：
1. 旅客票据
2. 驻留拖鞋
3. 牙刷与洗漱包
4. 气味卡
5. 环境参数卡
6. 植物记忆样本袋

## C. 让网页分层更明确
### 项目主页
- 面向策展人、合作方、普通观众
- 讲清项目是什么、为什么成立、怎么体验

### 档案查看器
- 面向 lore 用户和 ARG 用户
- 负责词条、组织、文件、异常记录

### 模拟官网层（下一轮）
- 作为“松间 / 白屿会”的表层网页
- 有真实感，但能逐渐漏出异常

## D. 让前期项目只做方法论铺垫
保留：
- New Media Artist Simulator
- CURATOR
- 一组空间视觉 / workshop / World Remix 参考

写法上不做“履历”，只做“方法前史”：
- 伪产品界面能力
- 关系型阅读体验
- 图形系统与空间视觉的积累

## E. 让气味和小物真正进入体验系统
不要把气味写成补充，而要写成流程的一部分：
- 扫码页可以问“你是否闻到某种植物性或冷空气气味？”
- 气味卡可作为现场带走物
- 洗漱套件和拖鞋可强化“旅居 / 入住 / 驻留”的生活层
