## 1. 命名修正（V2）

### 1.1 对外品牌
- **松间 / SONGJIAN**
- 性质：面向公众的旅居品牌 / 空间品牌
- 职责：赛尼亚岛的外层叙述、空间调性、地图、旅居物料、导览体验
- 说明：不再使用“松涧目的地 / PineBrook Destinations”。对外只保留“松间”这一单名品牌，气质更接近高端旅居系统，而不是地产说明文本。

### 1.2 背后开发与持有方
- **白屿会 / White Isle House**
- 性质：目的地持有方 / 开发与策划母体 / 类财团式背景结构
- 职责：目的地策略、空间持有、长期维护、岛屿资产与视觉秩序
- 说明：不再使用“北流空间开发集团”。新的名字更像一个真实存在的高端开发与策划母体，而不是直接写出“开发集团”。

### 1.3 航行与运输网络
- **潮线 / TIDELINE**
- 性质：交通运输 / 航行服务 / 节点通行网络
- 职责：旅客航行、目的地接驳、舱体运输、路线调度
- 说明：不再使用“数据洋流潜航署”。数字洋流作为底层媒介继续存在，但执行航行和运输的机构对外称为“潮线 / TIDELINE”。

### 1.4 异常物与回收系统
- **转栈署 / Relay Bureau**
- 性质：中转异常、错投物、遗失舱体与回收记录的管理机构
- 职责：登记、回收、归档、警示、失物编号与认领流程
- 说明：不再使用“空间转换机构”。新名字更简短，也更像一个真实世界中会留下文件痕迹的系统部门。

### 1.5 核心物件
- **项目题名：箱中温室 / Greenhouse in Transit**
- **物件名：雾庭舱 / Mistgarden Case**
- **内部型号：MG-17 Portable Conservatory Unit**
- 说明：项目在对外仍然叫《箱中温室》，但在世界观内部，这只手提箱不只是普通“箱子”，而是一类便携温室舱中的一个具体型号。这样它会更像《死亡搁浅》或《命运》里的核心装备，而不是单纯描述性标题。

## 3. 赛尼亚岛的层级位置

### 3.1 表层
赛尼亚岛数字旅居地是一个模拟挪威峡湾景观的虚拟北欧旅居地。用户首先接触到的是：
- 地图
- 建筑节点
- 路线
- 旅居空间
- Minecraft 可游玩入口
- “松间”品牌下的岛屿物料

### 3.2 深层
赛尼亚岛不是孤立景区，而是数字洋流中的一个稳定驻泊节点。它适合长期停留，也适合储存环境、缓冲记忆、承接旅客与研究者的停靠。

### 3.3 与《箱中温室》的关系
雾庭舱并不是岛屿本体，而是原本应该被带回赛尼亚岛内部某处空间的便携温室舱。它的错投，让赛尼亚岛第一次以“失物证据”的方式侵入现实。

## 6. 体验流程（V2）

### 6.1 线下总流程
```text
远看箱体 → 知道它是一件“遗失物” → 读到极少量说明
→ 扫码进入项目主页 → 理解赛尼亚岛 / 松间 / 雾庭舱
→ 进入档案查看器 → 发现转栈署记录与旅客线索
→ 佩戴 Vision Pro 看见雾庭舱展开成温室空间
→ 回到现实箱体继续看小物、票据、参数卡、气味卡
→ 带走纸质物，回家继续追踪网页更新
```

### 6.2 手机端四步
1. **识别页**：你看到的是一件异常登记中的便携温室舱。  
2. **公开页**：项目简介、赛尼亚岛、松间、地图。  
3. **异常页**：转栈署失物记录、旅客信息残片。  
4. **深入页**：档案查看器、Vision Pro 说明、后续 ARG 入口。  

### 6.3 Vision Pro 单次节奏（60–90 秒）
- 0–10 秒：箱体静置，边缘轻微异常
- 10–20 秒：票据与锁扣苏醒，边界线浮现
- 20–40 秒：雾庭舱内部球壳展开
- 40–65 秒：线框温室、点云植物、虹彩圆盘与代码层生长
- 65–85 秒：局部花园记忆显影，形成短暂空间包裹
- 85–90 秒：空间回收，留下未认领状态
