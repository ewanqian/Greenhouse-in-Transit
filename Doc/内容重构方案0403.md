# 一、项目总结构

## 1. 项目母标题

**箱中温室 / Greenhouse in Transit**

## 2. 项目一句话定义

一只来自数字航行系统的遗失手提箱，内部封存着一段植物性记忆温室；它因空间复制错误掉入现实，并通过实体物件、网页档案、手机端线索与头显观看逐渐展开。

## 3. 项目三层结构

### A. 对外展示层

给策展人、观众、合作方看的项目主页

### B. 世界观档案层

# 五、你这次最需要的体验流程

下面是我建议的完整版用户路径。

## 体验总流程

### Stage 0：远看

用户先看到一只箱子，知道它是本次唯一媒介之一。\
&#x20;这和展览整体规则匹配。

### Stage 1：近看

用户发现箱子不是单纯展示物，有标签、票据、物件、说明、气味、遗失感。

### Stage 2：读取

用户看到某个二维码 / 编号 / 失物说明。

### Stage 3：扫码

进入手机网页，不是直接跳长文，而是进入“失物识别页 / 档案入口页”。

### Stage 4：分流

用户可以选：

- &#x20;公开项目介绍&#x20;
- &#x20;失物认领入口&#x20;
- &#x20;旅居地地图&#x20;
- &#x20;档案阅读器&#x20;
- &#x20;乘客记录&#x20;

### Stage 5：深入

用户开始从表层“旅居品牌”进入深层“异常遗失”。

### Stage 6：扩展

部分用户继续进入：

- &#x20;Vision Pro 短体验&#x20;
- &#x20;假官网 / 假系统入口&#x20;
- &#x20;票据与物件扫描页&#x20;
