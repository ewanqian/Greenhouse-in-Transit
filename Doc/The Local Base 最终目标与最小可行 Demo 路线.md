# The Local Base 最终目标与最小可行 Demo 路线

更新时间：2026-04-14

## 1. 最终目标

《箱中温室 / Greenhouse in Transit》在当前阶段的最终目标，不是一次性做成完整的原生空间计算作品，而是先做出一条稳定、优雅、可展示、可继续扩展的最小可行链路：

- 现实里有一只成立的箱体
- 手机扫码后有一个清晰的项目入口页
- Vision Pro 里能看到一次完成度高的空间展开
- 线上网页、现场物件、头显体验讲的是同一件事

这条链路成立后，项目就已经足以作为：

- 展览现场的实体作品
- 艺术家项目说明页
- 后续 WebAR / 原生 visionOS / ARG 的共同底座
- 一个完整的案例仓库

## 2. 当前阶段定义

当前最适合走的不是“功能最全”，而是“The Local Base”路线：

- 先把底座搭稳
- 先用动画和少量三维资产打出气质
- 先让箱体、网页、Vision Pro 体验三者成立
- 高斯、复杂交互、自动识别、原生实时系统全部放到第二阶段

一句话说：

> 第一阶段不是做一个炫技 demo，而是做一个优雅的空间入口原型。

## 3. 本阶段最小可行 Demo

### 3.1 现实层

- 一个经过改造的手提箱
- 箱内少量核心物件：标签、参数卡、植物性样本、结构件、票据
- 箱体表面有编号、机构痕迹、遗失语义、轻微发光或反光细节

### 3.2 手机层

- GitHub Pages 上的项目主页
- 一个非常明确的扫码入口
- 不需要复杂 WebXR，先把说明页、档案页、认领页做顺

### 3.3 Vision Pro 层

- 一个 60–90 秒的空间展开动画
- 主资产来自 Blender 手 K 的几组状态
- 以 USDZ 资产和视频内容为主，不依赖重交互

## 4. 建议的体验链路

### 4.1 现场路径

1. 看见箱子
2. 发现它不是普通箱子
3. 扫码进入项目主页
4. 读到少量但有效的背景
5. 进入档案查看器或认领页
6. 佩戴 Vision Pro 看一次展开

### 4.2 不同入口的内容优先级

- 手机网页：负责解释和引导
- Vision Pro：负责 showoff 和空间显影
- 实体箱体：负责让故事先在现实里成立

## 5. 第一阶段美术与叙事目标

你这次不适合做“炫酷科技 UI”，而适合做一种更克制、更优雅、更像舞台调度的空间展开：

- 不是满屏赛博 HUD
- 不是重工业科幻
- 不是写实温室
- 也不是普通唯美花园

更合适的是：

- 温室作为被压缩的舞台装置
- 线框作为结构骨架
- 点云 / 植物作为记忆残影
- 虹彩圆盘 / 半透明片层作为光感控制面
- 漂浮代码 / 坐标杆 / 圆形轨道作为系统语言

一句话视觉方向：

> 这是一个被错误投递到现实的优雅温室，而不是一个喧闹的未来科技展台。

## 6. Vision Pro 动画分幕

### 幕 1：静置与感应（0–10 秒）

- 箱体静置
- 空间中只有很轻的呼吸感
- 一两处标签、灯点、编号轻微苏醒
- 不要一上来就爆开

### 幕 2：边界显影（10–25 秒）

- 圆形轨道、坐标杆、边界球壳慢慢显现
- 可以用线框、薄雾、半透明圆盘完成第一层空间感
- 这个阶段重点是“系统启动”，不是“花园爆发”

### 幕 3：结构展开（25–50 秒）

- 温室骨架从箱中向外生长
- 线框节点沿轨道形成支撑结构
- 几何壳体、薄片、支架、数据残影同步生长

### 幕 4：植物记忆显影（50–75 秒）

- 点云植物、高斯残影、局部有机体出现
- 可以是分层出现，而不是一次铺满
- 这里才开始出现“温室”而不是“设备”

### 幕 5：短暂停留与回收（75–90 秒）

- 空间达到最美状态后短暂停住
- 然后开始衰减或回收
- 留下“未认领 / 未完成 / 仍在中转”的尾音

## 7. Blender 最小制作方法

### 7.1 不追求复杂 rig

首版完全可以用手 K 做：

- 3 到 4 组关键状态
- 插值拉出优雅过渡
- 不依赖复杂模拟
- 不依赖复杂骨骼

### 7.2 适合先手 K 的资产

- 圆形轨道
- 坐标杆
- 线框球壳
- 简化温室骨架
- 虹彩圆盘
- 半透明片层
- 几组植物残影

### 7.3 动画节奏建议

- 每种元素只做 2 到 3 个变体
- 通过出现顺序和节奏差异拉出层次
- 避免所有物体一起动
- 让空间像舞台 cue 一样一层层亮起

### 7.4 材质策略

- 不做重烘焙
- 以发光、半透明、Fresnel、渐变和简单贴图为主
- 重点放在轮廓和层次，而不是写实细节

## 8. 技术路线结论

### 8.1 第一优先路线

- Blender 做动画资产
- 导出 USDZ 或拆分为多个 USDZ 资产
- 在 Reality Composer Pro / Xcode 里组装
- 以预设动画为核心

这条路线最符合当前目标：稳、快、可控。

### 8.2 第二优先路线

- 对部分难实时实现的段落，使用高质量视频承担 showoff
- 视频负责气氛和视觉密度
- 模型负责空间锚点和成立感

### 8.3 暂时不要作为首版主线的路线

- 把整个 demo 压在 PLY 高斯文件上
- 指望 WebXR 在所有设备上统一表现
- 一开始就做强交互识别系统

这些都容易拖慢首版交付。

目标是最小可运行 demo：

- 箱体实体成立
- 手机网页入口成立
- Vision Pro 有一版完整展开动画
- 可以向策展和合作方清楚说明项目价值

### 9.2 5 月 28 日前

目标是正式版增强：

- 优化视觉质量
- 补高斯 / 点云层
- 补 AR 延展入口
- 统一展陈和文案

## 10. 这轮之后最优先的执行清单

1. 定义首版 Vision Pro 动画只做哪 6 类资产
2. 画出分幕时间轴并锁定 60–90 秒脚本
3. 在 Blender 里先做 3 个关键状态的 blocking
4. 测试 USDZ / 视频在 Vision Pro 里的最小可行显示
5. 同步更新网页入口，让扫码逻辑更明确

## 11. 工作原则

- 先成立，再复杂
- 先优雅，再炫技
- 先让空间剧场成立，再做技术扩展
- 先做可展示版本，再做研究版本
